Easy ... To Oracle - Features and futures
Features
 Easy To Oracle (ETO) currently handles these types of datasources:
• Microsft SQL Server (TDS protocol)
• Sybase (TDS protocol)
• MySQL 5.x
• Microsoft Excel (all versions)
• OpenOffice/LibreOffice (all versions)
• SQLite 3.x
• Microsoft Access 97 - 2003 (discontinued in ETO 2.0 version)
• Presto (Hadoop)

• All datatypes are handled except: (var)binary, lob's, image. Fields with these datatypes are automatically filtered out by PL/SQL package.
Other fieldtypes are converted to corresponding Oracle types. Mostly to varchar2, number and date. User- defined conversions are possible, as long as they are valid for external tables. Character fields with content longer than 4,000 bytes are cut of at that length (extended varchar length of Oracle 12c is partially supported). Character set conversions are carried out by Oracle.

• ETO will accept the name of the source table or an SQL statement ( if the source database supports SQL).

• ETO can cache an arbitrary number of records in memory (user settable). On a multi-core processor it is possible to divide the workload between ETO and Oracle using this parameter.

• Oracle parallel dataloads (ie. multiple threads that read different parts from one table and combine the results) are not supported.
Running multiple dataloads at the same time is possible.

With PL/SQL package: UTL_ETO it is possible to:
• Define datasources
• Create an external table definition from the name of the source table
• Create multiple external table definitions at once for several or all tables in a remote database
